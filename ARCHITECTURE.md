# Архитектура проекта "Эммануэльевское урочище"

Данный проект ориентирован на современные стандарты React-разработки с использованием TypeScript (`.tsx`, `.ts`). Ниже описана структура папок и логика работы.

## Структура папок

```text
src/
├── components/          # Все React-компоненты приложения
│   ├── DiseaseCard/     # Компонент карточки заболевания (ОРВИ, Астма и т.д.)
│   ├── HomeScreen/      # Главный экран (список карточек заболеваний)
│   ├── LoadingScreen/   # Экран первоначальной загрузки (пульсирующее легкое)
│   └── MapScreen/       # Экран детального маршрута с картой 2ГИС
├── data.ts              # Хранилище текстовой информации (маршруты, реферат, координаты)
├── types.ts             # TypeScript-интерфейсы (строгая типизация данных)
├── App.tsx              # Главный компонент маршрутизации (управляет логикой экранов: Загрузка -> Главный -> Карта)
├── index.tsx            # Точка входа в React-приложение
├── index.css            # Глобальные стили и адаптивная верстка (Mobile + Desktop)
└── react-app-env.d.ts   # Служебный файл определения типов (для импорта CSS и модулей)
```

## Ключевые архитектурные решения

1. **Изолированные компоненты (Папочная структура)**  
   Вместо того чтобы хранить все компоненты в одном большом файле (джуниорский подход) или в виде простого списка фалов, каждый компонент имеет свою папку и файл `index.tsx`. 
   *Преимущества:* Это позволяет в будущем легко добавлять локальные стили (например, `DiseaseCard.module.css`), тесты (`DiseaseCard.test.tsx`) и дополнительные вложенные микро-компоненты, не засоряя корень папки `components`.

2. **Данные отделены от представления (Data Layer Separation)**  
   Вся информация (названия, краткие описания, полный реабилитационный потенциал) вынесена в `data.ts`. Компоненты (`HomeScreen`, `MapScreen`) ничего не знают о тексте — они только принимают данные через свойства (Props) по строгим правилам, описанным в `types.ts`. Это позволяет легко менять тексты без риска сломать логику интерфейса.

3. **Интеграция 2ГИС (API Layer)**  
   Поскольку 2ГИС блокирует стандартное встраивание ссылок по соображениям безопасности браузеров, используется динамическая загрузка **2GIS Maps API** (`loader.js`). Это позволяет аппаратно рендерить карту прямо в DOM дерева React. Логика этого API изолирована внутри компонента `MapScreen`.

4. **Адаптивный дизайн (Mobile-first CSS)**  
   В `index.css` изначально прописаны стили для мобильных устройств, так как приложение имитирует мобильный навигатор. Для компьютеров используются медиа-запросы (`@media (min-width: 768px)`), которые перестраивают макет (грид-сетки, разбиение на колонки) без дублирования HTML-кода.
